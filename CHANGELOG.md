# Changelog

## 2026.04.18

### 修复

- 新增退出保护诊断快照，统一记录权限快照、tap 快照、签名状态、sandbox 状态与推断出的失败原因
- 新增退出保护遥测封装，统一写入系统日志并同步落盘到 `Application Support/CQ/quit-guard-diagnostics.json`
- 抽出 event tap 策略，按显式策略顺序尝试创建 tap，并记录每次策略尝试结果
- 启动与唤醒编排改为基于“权限 + 环境 + tap”联合决策，避免签名异常或环境阻断时仍误判为 ready
- 修复 `AppLog` 统一日志写法，确保退出保护关键日志可以被 `log show --info --debug` 检索到

### 测试

- 补充签名无效阻断、唤醒环境阻断、tap 策略尝试记录、诊断文件落盘与降级写入的单元测试
- 调整目录路径断言与 tap 恢复失败测试桩，提升单测在本机环境下的稳定性

## 2026.04.11

### 修复

- 补充退出保护权限双检，区分辅助功能与输入监控 / 事件监听权限，并在权限不足时展示动态授权窗口
- 重构退出保护运行时封装，统一管理权限快照、运行状态与 `event tap` 生命周期
- 让 `event tap` 的创建失败、系统停用与恢复结果显式回传，避免 `Cmd+Q` 拦截静默失效
- 调整启动与唤醒后的退出保护编排，按当前权限状态决定是启动、重建还是停止拦截
- 测试环境下禁止触发授权提醒，避免运行测试时弹出辅助功能或事件监听授权窗口
- 补充退出保护链路的关键日志，便于后续定位权限缺失、`event tap` 创建失败与恢复异常

### 测试

- 补充退出保护权限服务、运行状态编排与 `event tap` 恢复路径的单元测试

## 1.5 · 2026.04.03

### 优化

- 更新应用图标与菜单栏图标资源
- 修复状态栏彩色图标被模板渲染成白色圆块的问题
- 调整配置持久化方案，通用配置改用 `UserDefaults`，白名单改为独立文件存储
- 修复首次启动时白名单文件不存在导致的 `cacheCQ.json` 读取报错
- 重绘菜单栏弹层界面，统一通用与白名单页的视觉风格和布局结构
- 优化弹层头部文案显示、内容区高度与页签切换时的留白稳定性
- 提高菜单栏弹窗与设置页高度，缩小内部字号和控件间距，整体更紧凑
- 移除内容区外层滚动，仅保留白名单列表在需要时滚动，避免默认情况下出现滚动条
- 优化白名单大数据量场景下的命中性能，并支持在面板内搜索应用名或路径
- 新增白名单应用内选择层，默认可直接在菜单弹窗内搜索并选择已安装应用，同时保留 Finder 手动选择兜底
- 修复菜单栏应用场景下添加白名单时系统选择器可能被压到底层、且 `.app` 选择不稳定的问题
- 调整状态栏图标交互，左键和右键单击都会切换弹窗
- 调整菜单栏弹窗生命周期管理，点击弹窗外部区域时会可靠收起
- 移除菜单中的更多设置按钮入口，保留其他设置窗口打开方式
- 重做 Cmd+Q 退出提示层，改为带标题、说明、快捷键徽标与停留进度的居中 HUD 卡片
- 复用退出提示窗口并补充结构化提示模型，减少重复弹出时的闪烁，同时兼顾长按忽略提示
- 补充菜单面板草稿配置写回与待应用状态的单元测试
- 补充白名单索引、搜索过滤与状态栏按钮事件的单元测试
- 补充白名单应用扫描、应用选择流转与菜单栏弹窗收起策略的单元测试
- 补充退出提示文案与退出保护状态流转的单元测试
